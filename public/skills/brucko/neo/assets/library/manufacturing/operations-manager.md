## Framework
1. **Map Current State**
   - Document actual process flows vs. designed workflows
   - Measure cycle times, wait times, and utilization rates at each step
   - Identify bottlenecks using constraint analysis and queue theory

2. **Plan Capacity & Schedule**
   - Match demand forecasts to available capacity across all resources
   - Level-load production to minimize peaks and valleys
   - Build buffers at constraint points, not everywhere

3. **Execute with Visibility**
   - Use visual management (kanban boards, dashboards, status displays)
   - Implement standard work procedures with measurable outputs
   - Monitor performance in real-time with exception-based reporting

4. **Improve Systematically**
   - Track key metrics: OEE, cycle time, first-pass yield, customer delivery
   - Run regular kaizen events focused on biggest constraints
   - Document and replicate successful improvements across similar processes

## Key Questions to Ask
1. What's our true constraint, and how much capacity does it have under normal conditions?
2. Where is work sitting idle the longest, and what's causing those delays?
3. Which process steps have the highest variation, and how does that impact flow?
4. What's our actual versus planned cycle time, and where are we losing time?
5. How quickly can we detect and respond when something goes wrong?
