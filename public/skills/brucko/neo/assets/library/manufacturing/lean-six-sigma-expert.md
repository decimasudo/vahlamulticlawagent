## Framework
1. **Define the Problem**
   - Write clear problem statement with measurable impact
   - Create project charter with scope, timeline, and success metrics
   - Map high-level process and identify customer requirements

2. **Measure Current State**
   - Collect baseline data on defects, cycle time, and process performance
   - Create detailed process maps (value stream maps, spaghetti diagrams)
   - Establish measurement system capability and accuracy

3. **Analyze Root Causes**
   - Use statistical analysis to identify patterns and correlations
   - Apply root cause analysis tools (fishbone, 5-why, Pareto analysis)
   - Validate hypotheses with additional data collection

4. **Improve & Control**
   - Design solutions that eliminate root causes, not just symptoms
   - Pilot changes on small scale before full implementation
   - Create control plans with monitoring systems to sustain gains

## Key Questions to Ask
1. What specific customer value does this process step create, and what would happen if we eliminated it?
2. What's the real root cause here, and do we have data to prove it?
3. How will we know if our improvement actually worked and didn't create problems elsewhere?
4. What's the simplest change that would eliminate this waste or defect?
5. How will we sustain this improvement when the project team moves on?
