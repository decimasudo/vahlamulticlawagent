## Framework
1. **Discovery** â€” Understand the problem space
   - Talk to actual users, not just stakeholders
   - Quantify the problem size and frequency
   - Map user journey pain points
   - Validate assumptions with data

2. **Strategy** â€” Define what to build and why
   - Set measurable success criteria upfront
   - Prioritize using impact/effort scoring
   - Create user stories with clear acceptance criteria
   - Align roadmap to business objectives

3. **Execution** â€” Ship iteratively and learn
   - Break big features into testable increments
   - Run experiments before full builds
   - Monitor leading indicators, not just vanity metrics
   - Communicate progress transparently

4. **Optimization** â€” Double down on what works
   - A/B test variations of successful features
   - Kill underperforming initiatives quickly
   - Scale wins across user segments
   - Document learnings for future decisions

## Key Questions to Ask
1. What specific user problem does this solve, and how do we know it's worth solving?
2. What does success look like in measurable terms, and when will we know?
3. What's the smallest version we can ship to learn if we're on the right track?
4. Who are our users actually, and have we talked to them recently?
5. What would have to be true for this to fail, and how do we test those assumptions?

| **Churn** | Rate at which customers stop using your product |
| **North Star Metric** | Single number that best captures the value you deliver |

## When to Apply
- Deciding what to build next or prioritizing roadmap
- Resolving conflicts between different stakeholder requests
- Designing experiments or A/B tests
- Analyzing user behavior or product metrics
