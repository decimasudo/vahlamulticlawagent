## 1. System Purpose

This library provides examiner-oriented, evidence-based cross-examination
arguments for challenging trademark use evidence in CNIPA non-use cancellation
proceedings.

It applies to:

- Applicant-side attack
- Respondent-side self-audit and pre-defense testing

Hard Constraints:

- No fabrication
- No speculation
- No emotional language
- Every point must be tied to:
  → A specific exhibit
  → A specific defect type
  → A verifiable factual basis

---

## 2. Mandatory Usage Rules

For each exhibit:

1. Identify Defect Type (A–F)
2. Select 1–3 Key Questions
3. Draft an objective closing clause
4. Reference:
   - Exhibit ID
   - Formation Date (if shown)
   - Carrier (contract / invoice / screenshot / etc.)

Generic attacks are prohibited.

## 3. Defect Classification Framework (A–F)

==================================================
A. Authenticity / Source Defects
真实性 / 来源瑕疵
==================================================

### A1. Original / Verification Missing

Standard Clauses:

- 该证据未提供原件或可核验来源信息，无法排除事后补制或内容修改的可能，真实性不足以采信。
- 该材料缺乏形成过程、原始载体或系统记录支撑，难以认定为交易过程中自然形成的真实凭证。

Key Questions:

- 是否提供原始文件及生成路径（系统导出/后台记录/日志）？
- 是否存在可独立核验的第三方记录？

---

### A2. Stamp / Signature Inconsistency

Standard Clauses:

- 证据所载签章、签名与其他材料明显不一致，且无合理解释，真实性存疑。
- 同一交易链条中编号规则、格式体系缺乏连续性，自然形成痕迹不足。

Key Questions:

- 是否存在授权链证明？
- 编号规则是否符合该主体长期使用习惯？

---

### A3. Self-Produced Evidence Without Corroboration

Standard Clauses:

- 该证据属于权利人单方制作材料，缺乏第三方印证，单独不足以证明真实商业使用。

Key Questions:

- 是否存在物流、税控、平台、支付流水等交叉验证材料？

==================================================
B. Relevance Defects
关联性瑕疵
==================================================

### B1. Goods / Services Unclear

Standard Clauses:

- 商品或服务描述不清，无法对应核定项目，关联性不足。
- 仅能证明存在交易或宣传行为，不能证明针对核定商品/服务。

Key Questions:

- 是否存在型号、SKU、清单或服务明细？
- 是否存在超范围或错位使用？

---

### B2. Mark Not Shown / Not Linked

Standard Clauses:

- 材料未体现争议商标，或缺乏与交易标的的绑定关系，不能构成使用证明。

Key Questions:

- 商标是否明确显示于包装、页面或凭证中？
- 是否与具体交易对象一致？

---

### B3. Subject Mismatch

Standard Clauses:

- 使用主体与注册人不一致，且无有效授权证明。

Key Questions:

- 是否提交许可合同及有效期证明？
- 实际控制关系是否成立？

==================================================
C. Temporal Defects
时间效力瑕疵
==================================================

### C1. Outside Use Period

Standard Clauses:

- 形成时间不在指定期间内，不能替代期间内使用证明。

Key Questions:

- 日期是否可核验？
- 是否存在时间断档？

---

### C2. Editable / Unverifiable Timestamp

Standard Clauses:

- 时间信息缺乏可核验性，存在编辑风险。

Key Questions:

- 是否有后台数据、存证或服务器日志支持？

==================================================
D. Completeness Defects
完整性瑕疵
==================================================

### D1. Single-Link Evidence

Standard Clauses:

- 仅证明单一环节，未形成闭环，证明力不足。

Key Questions:

- 是否存在订单、付款、交付对应关系？

---

### D2. Missing Annexes

Standard Clauses:

- 缺少附件、清单或履行材料，无法确认交易实质内容。

Key Questions:

- 是否提供完整履约文件？

==================================================
E. Logical / Commercial Defects
逻辑 / 合理性瑕疵
==================================================

### E1. Unreasonable Pattern

Standard Clauses:

- 交易模式与经营规模、渠道逻辑不匹配，合理性不足。

Key Questions:

- 是否存在持续往来记录？
- 是否符合行业惯例？

---

### E2. Circular / Internal Transactions

Standard Clauses:

- 更接近内部流转，缺乏真实市场属性。

Key Questions:

- 是否存在独立第三方支付与物流？

==================================================
F. Resistance Defects
抗反问能力瑕疵
==================================================

### F1. One-Question Collapse Test

Standard Clauses:

- 无法回答“谁、何时、就何商品/服务、如何使用”的核心问题，抗反问能力不足。

==================================================
4. Standard Closing Clauses
==================================================

Select one:

- 综上，该证据在真实性、关联性、时间效力及完整性方面存在明显缺陷，难以证明指定期间内真实使用。
- 综上，该组证据未形成可核验的商业使用闭环，不符合撤三案件证明标准。
- 综上，相关材料不足以证明商标使用行为在指定期间内真实发生，依法不应采信。
